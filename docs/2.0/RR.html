<static> context(key, value) → {type context}

context manages setting, getting, and persisting the recommendation context. The context defines user and page attributes that enable recommendations
Parameters:
Name Type Description
key {function} the key for a value to set or get; or an object that contains key value pairs to set
value {type context} value to set if key was passed. Optional

may return context
Type

Examples
 // set the product id
 RR.context({productId: '123abc'});
 
 // set the product id and name
 RR.context({productId: '123abc', productName: 'tic-tak'});
 
 // get the product id
 RR.context('productId');
 
 // get full context
 RR.context();
 
 // persist context
 RR.context('save');
 
 // get persisted context
 RR.context('load');

<static> fetch(callback, context) → {Object}

fetch mates a request to RR servers for dynamic content. Content will be returned as JSONP and send to a callback function passed to fetch. Additional context can also be passed to fetch, any values not passed with be inherited from the global context.
Parameters:
Name Type Description
callback {Function} the callback function for the request
context {Object} limited context for the request only, any values not set defer to global context. Optional
